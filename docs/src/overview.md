## 关键架构决策

### 1. 调度器算法：优先级位图（O(1)）
- 32 级优先级用 `uint32_t` 位图，`__builtin_clz` 取最高位
- 时间确定，满足硬实时要求
- 任务数有限时优先级表开销可忽略

### 2. 中间层：HAL + 统一驱动模型
- `arch/<cpu>/` 只放最少必要的 CPU 抽象
- `boards/<vendor>/<board>/` 负责板级差异
- 驱动通过 `device` 注册，使用者按名查找 —— 与 Linux 的 `device` 模型一致

### 3. 实时通信：集成而非自研
- 不自己实现 DDS，直接集成 **micro-ROS**（ROS 2 官方 MCU 版本）
- 优势：生态成熟、上位机互通
- 代价：增加 ~50KB ROM，学习曲线略陡

### 4. 内存策略：静态优先
- 默认禁止 `malloc` 进入内核路径
- 任务栈、消息缓冲、驱动 buffer 都用 `static` 或 `k_mem_pool`
- 动态分配作为可选模块，标记为「非确定性 API」

## 不做什么

明确划出**本期不做**的边界，避免范围蔓延：

- ❌ **MMU 支持**：聚焦 MCU 场景，不做虚拟内存
- ❌ **SMP 多核**：单核起步，多核作为后续扩展
- ❌ **POSIX 兼容层**：API 风格参考 POSIX 但不严格兼容
- ❌ **自研 DDS / TCP/IP 协议栈**：全部集成成熟第三方
- ❌ **安全认证**（如 IEC 61508）：需要时单独走流程
