# 需求规格说明书（Requirements）

> **状态**：v0.2（**scope 升级**：从 Cortex-M/MMU-less → Cortex-A + RV64 带 MMU）
> **对应阶段**：Phase 0 — 需求与范围定义
> **变更方式**：所有改动需在 [dev-log](../dev-log/) 留痕，重大改动需同步更新本文件

本文是整个项目的「宪法」。所有架构、API、驱动设计都必须能在本文中找到依据，反过来发现某条设计违反本文时，先改本文再改代码。

> ⚠️ **v0.2 重大变更**：硬件平台从「Cortex-M + RV32」升级到「**Cortex-A (AArch64) + RV64**」。
> 后果：**必须实现虚拟内存 / MMU**，原 v0.1 中「不做 MMU」已被推翻。
> 详细影响见 [dev-log/002](../dev-log/002-scope-upgrade-to-mmu.md)。

## 1. 硬件平台

### 1.1 CPU 架构

**必选**（异构双 ISA，逼出完整可移植层）：

| 架构 | ISA | 代表芯片 | 选择理由 |
|---|---|---|---|
| **ARM Cortex-A** | AArch64 | Raspberry Pi 4 (A72) / Pi 5 (A76) | 主流通用处理器、生态最广、能跑 Linux/ROS 2 完整工具链 |
| **RISC-V** | RV64GC | Sophgo SG2002 (C906) | 开源 ISA、面向未来、SG2002 是国产主流 RISC-V 64 SoC |

> **v0.2 关键变化**：
> - ARM 端从 Cortex-M4 改为 Cortex-A72/A76（AArch64）
> - RISC-V 端从 RV32IMAC 改为 RV64GC（lp64d ABI）
> - **两者都带 MMU**，虚拟内存从「可选」变成「必做」

### 1.2 参考开发板

- **ARM**：Raspberry Pi 4 Model B / Raspberry Pi 5（任选其一，Phase 2 之后定型）
- **RISC-V**：SG2002 开发板（如 **Lichee Pi 4A** / **Milk-V Duo** / **SG2002 评估板**）

### 1.3 仿真平台

QEMU 是主线开发/调试环境，**两个架构都用 QEMU**：

| 架构 | QEMU 命令 | 备注 |
|---|---|---|
| ARM | `qemu-system-aarch64 -machine raspi4b -kernel ...` | Pi 4 / Pi 5 模型均支持 |
| RISC-V | `qemu-system-riscv64 -machine virt -cpu rv64 -bios none` | SG2002 无原生 QEMU 模型，用 `virt` 通用 RISC-V 64 板仿真（需注明偏差） |

> **真机烧录** 留到 Phase 2 之后；QEMU 跑通是真机前唯一硬门槛。

## 2. 实时性指标

> 数字决定了调度器、上下文切换、MMU 配置的设计。**越紧越难实现**，不要随便改。
> v0.2 因为有了 MMU，部分指标从「纳秒级裸金属」变成「受 TLB / cache 影响」，测量方法更复杂。

| 指标 | 目标值 | 测量方法 |
|---|---|---|
| 中断响应时间（GIC/PLIC 入口到首条指令） | < 10 μs | GPIO 翻转 + 示波器（裸模式）/ 逻辑分析仪 |
| 任务切换时间（exception 入口到出口） | < 20 μs | 桩函数 + GPIO 翻转（MMU 开销包含在内） |
| Tick 周期（默认） | 1 ms | 配置宏 `TICK_RATE_HZ=1000` |
| Tick 周期（高精度模式） | 100 μs | 配置宏 `TICK_RATE_HZ=10000` |
| WCET（每 API） | 文档化 | Tracealyzer / 手动插桩 |
| **内核 RAM** | < 256 KB | 内核专用静态分配，不含用户态堆 |
| **内核 ROM** | < 1 MB | 内核镜像（含 .text / .rodata / 启动 stub） |
| **用户可用 RAM** | < 数 GB | 受 SoC 限制（Pi 4 = 1/2/4/8 GB，SG2002 = 256 MB） |

> **v0.2 变化**：
> - 任务切换时间放宽到 20 μs（MMU 上下文切换有 TLB 维护成本）
> - RAM/ROM 上限 **针对内核本身**，不再限制用户态可用内存
> - 数字仍偏紧，是「达标即合格」的目标，不是「做不到就改需求」

## 3. 调度策略

| 策略 | 是否支持 | 备注 |
|---|---|---|
| 固定优先级抢占式 | ✅ 主调度 | 必选 |
| 同优先级时间片轮转 | ✅ | 必选 |
| 优先级继承协议（PIP） | ✅ 互斥锁 | 必选 |
| 最早截止优先（EDF） | ⚪ 插件 | v1.1+ |
| 速率单调（RMS） | ⚪ 仅分析工具 | 不进调度器 |
| **SMP 多核调度** | ⚠️ 重新评估 | v0.2 决定：**v1.0 不做 SMP**，但 v2.0 路线保留 |

**任务数上限**：默认 64（v0.2 从 32 提高，64-bit 下 TCB 体积增大但 RAM 充足）。

## 4. API 风格

参考 FreeRTOS / Zephyr 的「名词_动词」式（**v0.2 无变化**）：

```c
task_t *task_create(const char *name,
                    void (*entry)(void *),
                    void *arg,
                    size_t stack_size,
                    int priority);
void task_yield(void);

sem_t *sem_create(int initial_count, int max_count);
int    sem_take(sem_t *sem, tick_t timeout);
int    sem_give(sem_t *sem);
int    sem_give_from_isr(sem_t *sem);
```

每个公开 API 必须文档化：

- 参数、返回值
- **是否可在 ISR 中调用**
- 时间复杂度（`O(1)` / `O(n)`）
- 阻塞语义
- **v0.2 新增**：**内存影响**（是否触发 page fault / TLB shootdown / 跨核 IPI）

## 6. 不做什么（Out of Scope）

v0.2 **重大变化**：

| 项 | v0.1 | v0.2 | 原因 |
|---|---|---|---|
| ❌ MMU / 虚拟内存 | 是 | **✅ 必做** | Cortex-A + RV64 都带 MMU，不做等于不用硬件 |
| ❌ SMP 多核 | 是 | 是（维持） | v1.0 仍不做，留 v2.0；但本版本已为此预留扩展点 |
| ❌ POSIX 严格兼容 | 是 | 是 | 维持 |
| ❌ 自研 DDS / TCP/IP | 是 | 是 | 维持 |
| ❌ 安全功能认证 | 是 | 是 | 维持 |
| ❌ 动态加载 | 是 | 是 | 维持 |
| ❌ 用户态进程隔离 | n/a | 是 | v1.0 单地址空间，所有 task 共享内核页表；进程隔离留 v2.0 |
| ❌ 设备虚拟化（IOMMU） | n/a | 是 | v1.0 设备直驱；v2.0 评估 IOMMU |

## 7. 内存模型（**v0.2 新增**）

这是 v0.1 完全没考虑的内容，现在必须明确。

### 7.1 地址空间布局（单地址空间，内核/任务共享）

```
虚拟地址空间（48 位，canonical）
┌─────────────────────────────────────┐ 0xFFFF_FFFF_FFFF
│   内核高地址（直接映射设备等）        │
│   ...                                │
│   0xFFFF_0000_0000                  │ ← 内核基址
├─────────────────────────────────────┤
│   内核栈 / 调度器 / 内核 BSS         │
├─────────────────────────────────────┤
│   内核直接映射区（设备 MMIO 1:1）    │
├─────────────────────────────────────┤
│   ...                                │
│   0x0000_8000_0000                  │ ← 用户基址（v1.0 暂未使用）
├─────────────────────────────────────┤
│   各任务的虚拟栈（按需分页）          │
│   共享库 / 共享内存（v2.0+）          │
├─────────────────────────────────────┤
│   0x0000_0000_0000                  │ ← 零页（未映射，防 NULL deref）
└─────────────────────────────────────┘
```

### 7.2 页表

- 粒度：**4 KB**（v1.0），2 MB hugepage v2.0 再议
- 算法：**Sv48**（RISC-V）/ **4-Level Translation Table**（ARMv8）
- 内核页表：启动时建好，单页表覆盖所有内存（v1.0 单地址空间）
- 任务页表：v1.0 **不单独建**（与内核共享），v2.0 引入

### 7.3 TLB 管理

- 全局映射（内核）→ 不需要 ASID
- 用户页（v2.0+）→ 16 位 ASID（避免频繁 TLB flush）
- 跨核 TLB shootdown → v1.0 不需要（单核），v2.0 用 IPI
