# 简介

> ⚠️ **v0.3.1 升级中**：本书正文（除 README 与 Phase 0 指针）仍反映 v0.1/v0.2 的 RTOS 路线，将在 Phase 1 启动时全面重写为双 ISA 统一混合内核版本。当前生效的设计见 [requirements v0.3.1](../requirements/requirements.md)。

本书是「用 C 语言从零为机器人构建一个多平台实时操作系统（RTOS）」的实战路线图。

## 为什么写这本书

市面上已经有 FreeRTOS、Zephyr、NuttX、RT-Thread 等成熟的 RTOS，并且支持大量机器人场景。但当出现以下需求时，从零实现仍有价值：

- **深度学习**：通过手写调度器、上下文切换、同步原语，彻底理解 RTOS 的每一行代码为什么这样写。
- **极致定制**：机器人项目常常需要裁剪到 KB 级 ROM/RAM、追求确定的最坏执行时间，或者把特定算法卸载到硬件。
- **特定硬件**：新 MCU 没有现成 RTOS 支持时，需要从零移植甚至重写。
- **不依赖第三方**：许可证、供应链安全、长期维护可控性。

## 本书路线图的三个关键选择

为了让范围可控，本路线图固定了以下选择（详见 [Phase 0](phase-0-requirements.md)）：

| 维度 | 选择 | 影响 |
|---|---|---|
| 起点 | **从零手写内核** | 工作量大但可控、深度学习价值高 |
| 平台 | **多平台支持**（ARM Cortex-M + RISC-V 起手） | 必须先做扎实的 HAL 层 |
| 功能 | **完整生态**（基础内核 + 驱动 + micro-ROS + FS + 网络 + GUI） | 工作量按 1–2 人年估算，需分阶段交付 |

## 阅读建议

- **第一次读**：先通读 [总览](overview.md) 与 [Phase 0](phase-0-requirements.md) ~ [Phase 3](phase-3-scheduler.md)，把握架构骨架。
- **动手前**：完成 [Phase 0](phase-0-requirements.md) 的需求文档、确认硬件清单。
- **每个 Phase 的末尾**都列出了「验证标准」——把它当作进入下一 Phase 的硬门槛。

## 范围提醒

完整覆盖勾选的全部功能是 **5–10 人年**的工作量。建议前 3 个月聚焦 Phase 1–4 + Phase 6 基础驱动，跑通「能控制电机 + 读编码器 + 实时通信」的最小机器人，再向外扩展。**避免在没有验证调度器之前先做 GUI**。
